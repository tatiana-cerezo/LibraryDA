# LIBRARYDA - PLAN DE PROYECTO
El objetivo del proyecto es planificar y organizar el desarrollo de un sistema de biblioteca, asegurando el cumplimiento de los hitos establecidos.

## CRONOGRAMA
El desarrollo del proyecto, consistente en la presentación de 5 hitos, se ha dividido en 3 fases diferenciadas:
-	Fase 1 (del 13 al 27 de enero): Desarrollo del MVP y documentación técnica.  
    - Diseño de la arquitectura principal del proyecto.  
    - Implementación del backend.  
    - Implementación del frontend (Thymeleaf).  
    - Elaboración de la documentación técnica: Modelo Entidad-Relación (ER), Diagrama de Clases UML, JavaDoc y API.

-	Fase 2 (del 27 de enero al 3 de febrero): Integración, pruebas y refinamiento.  
    - Pruebas funcionales y de integración.   
    - Revisión de seguridad y buenas prácticas.   
    - Implementación de requisitos transversales (i18n, RGPD y cookies).   
    - Pulido de errores.   
    - Actualización de documentación.

-	Fase 3 (del 3 de febrero al 10 de febrero): Despliegue.  
    - Pruebas finales de calidad.   
    - Documentación completa (técnica y manual de usuario).   
    - Configuración final.

## RECURSOS
-	Humanos:  
    - Desarrollador: responsable del análisis, diseño, desarrollo, pruebas y documentación del proyecto.

-	Tecnológicos:  
    - Lenguajes: Java, HTML, CSS, JavaScript.  
    - Frameworks: Spring Boot, Thymeleaf.  
    - Base de datos: MySQL.  
    - Herramientas de gestión: GitHub, Trello.  
    - IDE: IntelliJ IDEA / VS Code.

## ROLES
-	Desarrollador full stack: análisis, diseño, desarrollo, pruebas y documentación.

## INDICADORES DE ÉXITO
- Cumplir con todos los hitos en las fechas establecidas. 
-	Implementar todas las funcionalidades requeridas (CRUD libros, usuarios, préstamos). 
-	Relaciones *OneToMany* funcionando correctamente. 
-	Autenticación y autorización operativas. 
-	Multiidioma (español/inglés) funcional. 
-	Requisitos legales implementados (aviso legal, cookies). 
-	Aplicación ejecutándose con la configuración estándar.

## SUPOSICIONES Y DEPENDENCIAS
-	Suposiciones:  
    - Se dispone de un entorno local con las herramientas necesarias instaladas.   
    - El desarrollador tiene conocimientos de Spring Boot, JPA y Thymeleaf.   
    - Se cuenta con conexión a Internet para acceder a GitHub y dependencias Maven. 

-	Dependencias:  
    - MySQL instalado y operativo.   
    - Acceso al repositorio GitHub.   
    - Disponibilidad de las librerías y dependencias de Spring Boot.
